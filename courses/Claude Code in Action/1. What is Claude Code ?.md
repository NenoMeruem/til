# Giới thiệu khóa học

Khóa học này cung cấp chương trình đào tạo toàn diện về cách sử dụng **Claude Code** trong các tác vụ phát triển phần mềm. Nội dung bao gồm:
- Kiến trúc nền tảng (underlying architecture) của AI coding assistant
- Kỹ thuật triển khai (implementation techniques) trong môi trường thực tế
- Chiến lược tích hợp nâng cao (advanced integration strategies)

Bạn sẽ tìm hiểu cách **Claude Code** quản lý _context_ (ngữ cảnh làm việc), cũng như cách mở rộng chức năng thông qua **MCP server** và tích hợp với GitHub để tối ưu hóa workflow phát triển phần mềm.

## Bạn sẽ học được gì

### 1. Hiểu về kiến trúc của Coding Assistant

- Nắm được cách AI assistant tương tác với **codebase** thông qua cơ chế _tool integration_.
- Hiểu nền tảng kỹ thuật cho phép hệ thống phân tích (code analysis), sinh mã (code generation) và chỉnh sửa mã (code modification).
- Làm rõ vai trò của các thành phần như parser, abstract syntax tree (AST), và execution environment trong việc xử lý mã nguồn.
### 2. Khai thác hệ thống Tool của Claude Code
- Tìm hiểu cách Claude Code điều phối và kết hợp nhiều tool để xử lý các tác vụ lập trình nhiều bước (multi-step tasks).
- Ứng dụng trong các tình huống như refactor, debugging, test generation và feature implementation.
- Hiểu cách orchestration giữa các tool giúp tự động hóa quy trình phát triển.

### 3. Làm chủ kỹ thuật Context Management
- Học cách duy trì _relevant context_ xuyên suốt hội thoại với AI.
- Biết cách tham chiếu đúng tài nguyên trong project (source files, config files, documentation).
- Tối ưu hóa prompt để giảm nhiễu (noise) và tăng độ chính xác của phản hồi.
### 4. Triển khai quy trình giao tiếp trực quan (Visual Communication Workflow)
- Sử dụng input dạng hình ảnh (UI mockups, screenshot, wireframe) để mô tả thay đổi giao diện.
- Tận dụng các tính năng planning nâng cao để xử lý thay đổi trên codebase lớn.
- Kết hợp giữa phân tích cấu trúc UI và cập nhật logic backend khi cần.

### 5. Xây dựng Custom Automation
- Tạo các custom command có thể tái sử dụng.
- Thiết lập automation pipeline để xử lý các tác vụ lặp lại như linting, formatting, migration script.
- Chuẩn hóa workflow nhằm tăng năng suất và giảm lỗi thủ công (human error).

### 6. Mở rộng chức năng với MCP Server
- Tích hợp các công cụ và dịch vụ bên ngoài thông qua MCP server.
- Kích hoạt các khả năng nâng cao như browser automation, API orchestration và các workflow chuyên biệt.    
- Thiết kế kiến trúc mở rộng (extensible architecture) để Claude Code hoạt động như một thành phần trong hệ sinh thái DevOps.

### 7. Tích hợp với GitHub Workflow
- Thiết lập quy trình automated code review.
- Tích hợp AI vào pipeline quản lý mã nguồn sử dụng Git.
- Áp dụng trong pull request review, CI/CD, và branch management.
### 8. Áp dụng Thinking & Planning Modes
- Hiểu khi nào nên sử dụng các chế độ suy luận khác nhau tùy theo độ phức tạp của bài toán.    
- Phân biệt giữa tác vụ đơn giản (quick patch, minor fix) và bài toán kiến trúc phức tạp (system redesign, large-scale refactor).
- Tối ưu chiến lược giải quyết vấn đề dựa trên complexity level và scope của project.
## Yêu cầu trước khi tham gia
- Thành thạo sử dụng command-line interface (CLI) và thao tác trên terminal.
- Có kiến thức cơ bản về version control với Git (commit, branch, merge, pull request).
## Khóa học này dành cho ai?
- Lập trình viên muốn tích hợp AI assistant vào workflow phát triển phần mềm.
- Các team phát triển phần mềm muốn triển khai tích hợp GitHub có hỗ trợ AI cho nhiều quy trình khác nhau.

# What is a coding assistant?

Một **coding assistant** (trợ lý lập trình) không chỉ đơn thuần là một công cụ sinh ra các đoạn mã nguồn. Thực chất, nó là một **hệ thống phần mềm phức tạp** được xây dựng dựa trên **mô hình ngôn ngữ (Language Model)**, có khả năng phân tích, suy luận và xử lý các bài toán lập trình ở mức độ cao.

Dưới đây là bản dịch sử dụng các thuật ngữ chuyên ngành trong lập trình, nhưng được diễn đạt theo cách dễ hiểu và dễ tiếp cận:

## How Coding Assistant Work

Khi bạn giao cho một coding assistant (trợ lý lập trình) một nhiệm vụ — ví dụ như **debug một bug dựa trên error message** — nó sẽ thực hiện một quy trình tương tự như cách một developer xử lý vấn đề trong thực tế.

![Image](images/What%20is%20Claude%20Code/Pasted%20image%2020260224110710.png)


 1. Thu thập ngữ cảnh (Gather Context): Phân tích **error message** để hiểu lỗi đang nói đến vấn đề gì, xác định phần nào trong **codebase** bị ảnh hưởng, tìm các **file liên quan** đến lỗi.
2. Xây dựng phương án xử lý (Formulate a Plan): Quyết định cách sửa lỗi (ví dụ: refactor một đoạn code, thay đổi logic xử lý, hoặc cập nhật dependency), lên kế hoạch kiểm chứng kết quả bằng cách chạy test (unit test, integration test, v.v.).
3. Thực thi giải pháp (Take Action):  Chỉnh sửa file (modify source files), thực thi các command cần thiết (ví dụ: build project, chạy test suite, lint code)

Điểm Quan Trọng Cần Lưu Ý:  **Bước 1 (thu thập ngữ cảnh)** và **Bước 3 (thực thi giải pháp)** đều yêu cầu assistant phải **tương tác với môi trường bên ngoài (external environment)** - Đọc file trong hệ thống, truy xuất tài liệu (documentation), thực thi command trong terminal, chỉnh sửa mã nguồn.

## The Tool Use Challenge

Đây là lúc mọi thứ trở nên thú vị. Về bản chất, **language model (LLM – Large Language Model)** chỉ có khả năng **xử lý chuỗi văn bản (text input)** và sinh ra **chuỗi văn bản đầu ra (text output)**. Chúng **không có quyền truy cập trực tiếp vào hệ thống file (file system)**, **không thể thực thi lệnh (execute command)**, cũng không thể tương tác với môi trường runtime. Ví dụ:  Nếu bạn yêu cầu một LLM thuần túy đọc nội dung một file, nó sẽ phản hồi rằng nó **không có capability (khả năng hệ thống)** để thực hiện thao tác đó.

Vậy làm thế nào các **coding assistant** (trợ lý lập trình) giải quyết được bài toán này? Câu trả lời nằm ở một cơ chế thông minh gọi là **Tool Use**.

## How Tool Use Works

Khi bạn gửi một request đến coding assistant, hệ thống **không chỉ chuyển nguyên văn câu hỏi của bạn cho LLM**. Thay vào đó, nó sẽ tự động **Inject thêm instruction (chỉ thị hệ thống)** vào prompt, dạy LLM cách “yêu cầu hành động” thông qua một format chuẩn. Ví dụ, hệ thống có thể tự động thêm vào prompt một hướng dẫn như sau: “Nếu bạn muốn đọc file, hãy phản hồi theo format:  `ReadFile: <tên_file>`”

 Toàn bộ luồng xử lý (Execution Flow)
1. Bạn gửi yêu cầu: “Code trong file `main.go` là gì?”
2. Coding assistant sẽ **augment prompt** bằng cách thêm các tool instruction vào request ban đầu.
3. LLM phản hồi: ``ReadFile: main.go``
4. Coding assistant thực thi hành động thực tế: đọc file `main.go` từ file system và gửi nội dung file ngược lại cho LLM.
5. LLM nhận được nội dung file và sinh ra câu trả lời cuối cùng dựa trên dữ liệu thực tế.

Hệ thống này cho phép các LLM có thể “đọc tệp”, “viết mã” và “chạy lệnh” một cách hiệu quả, mặc dù thực chất chúng chỉ đang tạo ra các phản hồi văn bản được định dạng một cách cẩn thận.

> Tool Use là một cơ chế **orchestration giữa LLM và môi trường thực thi**, trong đó LLM đóng vai trò bộ máy suy luận (reasoning engine), còn hệ thống bên ngoài đảm nhiệm phần tương tác với tài nguyên hệ thống.

# Why Claude's Tool Use Matters

Không phải tất cả các mô hình ngôn ngữ đều giỏi như nhau trong việc sử dụng công cụ. Dòng mô hình Claude (Opus, Sonnet và Haiku) đặc biệt mạnh trong việc hiểu các công cụ làm gì và sử dụng chúng một cách hiệu quả để hoàn thành những nhiệm vụ phức tạp.

![Image](images/What%20is%20Claude%20Code/Pasted%20image%2020260225135658.png)


Khả năng sử dụng công cụ mạnh mẽ này mang lại một số lợi ích quan trọng cho Claude Code

## Benefits of Strong Tool Use

- **Xử lý các tác vụ khó hơn** – Claude có thể kết hợp nhiều công cụ khác nhau để giải quyết các công việc phức tạp và thậm chí sử dụng những công cụ mà trước đó chưa từng gặp.
- **Nền tảng có thể mở rộng** – Bạn có thể dễ dàng bổ sung công cụ mới vào Claude Code, và Claude sẽ thích nghi để sử dụng chúng khi quy trình làm việc của bạn phát triển.
- **Bảo mật tốt hơn** – Claude Code có thể điều hướng trong các codebase mà không cần lập chỉ mục (indexing), điều này thường đồng nghĩa với việc không phải gửi toàn bộ mã nguồn của bạn lên các máy chủ bên ngoài.

## Key Takeaways

Việc hiểu về các trợ lý lập trình có thể tóm gọn trong một vài điểm cốt lõi:

- Trợ lý lập trình sử dụng các mô hình ngôn ngữ để hoàn thành nhiều tác vụ khác nhau.
- Các mô hình ngôn ngữ cần công cụ để xử lý hầu hết các tác vụ lập trình trong thế giới thực.
- Không phải tất cả các mô hình ngôn ngữ đều có cùng mức độ thành thạo trong việc sử dụng công cụ.
- Khả năng sử dụng công cụ mạnh mẽ của Claude cho phép tăng cường bảo mật, khả năng tùy chỉnh và tính bền vững lâu dài trong Claude Code.

Chính khả năng sử dụng công cụ này đã biến một mô hình chỉ tạo văn bản đơn thuần thành một trợ lý lập trình mạnh mẽ, có thể đọc tệp của bạn, hiểu toàn bộ hệ thống mã nguồn và thực hiện những thay đổi có ý nghĩa cho các dự án của bạn.

# Claude Code In Action

Claude là một chuyên gia trong việc sử dụng tool (tool orchestration) và rằng Claude Code có khả năng mở rộng (extensible architecture) rất dễ dàng. Có thể bạn sẽ hơi hoài nghi, nên tôi muốn đưa ra một vài demo nhanh để chứng minh điều đó.

## Default toolkit in Claude Code.

Đây là danh sách các **default tools** có sẵn trong Claude Code. Nó có đầy đủ những khả năng mà bạn kỳ vọng ở một coding agent:

- Đọc file (file system read)
- Ghi file (file system write)
- Thực thi lệnh terminal (command execution)
- Và các thao tác hệ thống khác

![Image](images/What%20is%20Claude%20Code/Pasted%20image%2020260225135601.png)

Tôi sẽ đưa cho Claude Code một vài task. Trong mỗi task, nó sẽ sử dụng tập công cụ này theo cách khá thông minh — đúng kiểu một hệ thống có khả năng **reasoning + tool chaining** thực thụ. Ở ít nhất một task, tôi còn cấp thêm cho Claude một bộ tool mới để mở rộng năng lực của nó.
Qua quá trình này, bạn sẽ thấy:
1. Claude Code có thể làm được gì ngay khi “out of the box”.

Nghe có vẻ đơn giản và hơi “vô thưởng vô phạt”, nhưng thực tế thì đây là package được tải nhiều thứ 5 trong toàn bộ hệ sinh thái JavaScript. Điều đó có nghĩa là nó được sử dụng cực kỳ rộng rãi. Nếu tối ưu được dù chỉ một chút hiệu năng, lợi ích cộng dồn trên quy mô toàn hệ sinh thái sẽ rất lớn.

## Github Integration

![Image](images/What%20is%20Claude%20Code/Pasted%20image%2020260225153732.png)

Hãy xem ngay một ví dụ cụ thể.

**Claude Code** có khả năng tích hợp rất chặt chẽ với GitHub. Bạn có thể cấu hình Claude Code chạy bên trong một **GitHub Action**, nơi nó sẽ được thực thi tự động dựa trên các sự kiện (event trigger) như khi tạo **Pull Request (PR)** hoặc khi được mention trực tiếp trong một **Issue**.

Khi Claude Code chạy trên GitHub, nó không chỉ có quyền truy cập để đọc và thực thi source code của bạn, mà còn có thể sử dụng một bộ công cụ (tooling) chuyên biệt để tương tác với GitHub, ví dụ như:

- Tạo comment trong Pull Request
- Tạo commit
- Tạo hoặc chỉnh sửa Pull Request
- Và nhiều thao tác automation khác

Một trong những use case mạnh mẽ nhất của integration này là **tự động review Pull Request**.
  
### Ví dụ thực tế: Phát hiện rò rỉ dữ liệu nhạy cảm

Giả sử chúng ta đang xây dựng hệ thống hạ tầng (infrastructure) trên Amazon Web Services. Toàn bộ infrastructure được định nghĩa dưới dạng **Infrastructure as Code (IaC)** bằng các file Terraform và được commit, lưu trữ trên GitHub.

Vì tất cả tài nguyên hạ tầng đều được định nghĩa bằng Terraform, Claude Code có thể phân tích cấu trúc hệ thống và hiểu rõ **data flow** trong toàn bộ infrastructure.

    - `plans_viewed`
    - `registration_date`

Chúng ta muốn chia sẻ hai trường dữ liệu này cho:
- Một team marketing nội bộ
- Một đối tác marketing bên ngoài
Để làm điều đó, mỗi đêm một hàm AWS Lambda sẽ:
1. Query toàn bộ user mới trong DynamoDB
2. Extract hai field `plans_viewed` và `registration_date`
3. Ghi dữ liệu vào một bucket trên Amazon S3

Hai team marketing sẽ truy cập dữ liệu thông qua S3 bucket này.

### Vấn đề phát sinh

Vài tháng sau, team marketing nội bộ yêu cầu thêm trường `email` vào dữ liệu xuất ra S3.
Bạn vào Lambda function và thêm đúng **một dòng code** để đưa `user.email` vào payload ghi xuống S3 bucket.
Vấn đề là:
Sau vài tháng, bạn có thể đã quên rằng S3 bucket này **được chia sẻ với đối tác bên ngoài**.
Điều đó đồng nghĩa với việc bạn đang vô tình ghi **PII (Personally Identifiable Information)** – cụ thể là email – vào một bucket mà công ty khác có quyền truy cập.
Đây là một lỗi nghiêm trọng về bảo mật và tuân thủ (compliance violation).
Và thực tế, loại lỗi này hoàn toàn có thể xảy ra trong môi trường production nếu không có cơ chế kiểm soát tốt.


![Image](images/What%20is%20Claude%20Code/Pasted%20image%2020260225161117.png)

### Claude Code phát hiện vấn đề như thế nào?
Vì toàn bộ infrastructure được định nghĩa bằng Terraform, Claude Code có thể:
- Phân tích cấu hình hạ tầng (infrastructure graph)
- Hiểu quan hệ giữa:
    - DynamoDB
    - Lambda
    - S3
    - Quyền truy cập của external partner
- Theo dõi data flow từ database đến S3 bucket

Khi bạn tạo Pull Request thêm trường `email`, Claude Code trong quá trình **automated PR review** đã:
1. Phân tích toàn bộ diff trong Pull Request
2. Mapping thay đổi code với kiến trúc hạ tầng
3. Phát hiện rằng bạn đang expose PII cho đối tác bên ngoài
4. Liệt kê chi tiết data flow:
    - Dữ liệu từ DynamoDB
    - Qua Lambda
    - Được ghi vào S3
    - Và bucket này có external access
Việc phát hiện lỗi **ngay trong giai đoạn code review** thay vì sau khi deploy lên production là một lợi ích cực kỳ lớn.

Claude Code hoạt động hiệu quả như vậy nhờ khả năng sử dụng công cụ (tool usage) và phân tích ngữ cảnh toàn diện của repository. Bạn nên xem Claude Code như một **AI assistant linh hoạt**, có thể tùy biến để tích hợp sâu vào workflow CI/CD và DevOps của bạn, thay vì chỉ là một chatbot đơn thuần. Ở phần sau, chúng ta sẽ đi chi tiết cách cấu hình một workflow tương tự để bạn có thể triển khai cơ chế automated infrastructure-aware code review này trong dự án của mình.
